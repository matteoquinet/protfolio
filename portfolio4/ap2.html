Atelier Professionnalisation – Projet CashCash

De Septembre à Avril 2026 – Volume horaire : 100h en atelier + travail personnel

Présentation de l'entreprise CashCash

CashCash est une entreprise spécialisée dans la vente de materiels informatiques : caisses automatiques, lecteurs de cartes, écrans, souris, etc, pour des commercres.

CashCash possède un système d'information (SI) pour Linux et Windows.

Dans le cadre de cet atelier, nous sommes salariés de VDEV, une société prestataire chargée de répondre aux besoins informatiques de CashCash.

Objectif de l'atelier

L'objectif principal de cet atelier est de développer une application web et un client lourd pour la gestion des interventions techniques et des contrats de maintenance chez CashCash.

Le projet est organisé en jalons :

  • Jalon 1 : Analyse, planification et mise en place des outils collaboratifs.
  • Jalon 2 : Développement de l'application web (gestion des interventions).
  • Jalon 3 : Développement du client lourd (gestion des contrats et export XML/PDF).

Jalon 2 : Développement de l'Application Web

Le Jalon 2 a pour objectif de développer une application web responsive pour la gestion des interventions techniques chez CashCash.

Fonctionnalités développées :

  • Authentification sécurisée : Mots de passe forts, cryptage et hachage.
  • Gestion des interventions :
    • Recherche de fiches clients par numéro.
    • Génération de fiches d'intervention au format PDF.
    • Consultation des interventions par date ou par agent.
    • Validation des interventions avec commentaires et temps passé.
  • Tableau de bord pour les gestionnaires :
    • Affectation des visites aux techniciens.
    • Outil statistique pour visualiser le nombre d'interventions, les kilomètres parcourus et le temps passé par matériel.

Contraintes techniques :

  • Architecture MVC.
  • Responsive design (compatible avec tous les terminaux et navigateurs).
  • Utilisation de JavaScript, Ajax, JQuery.
  • Base de données avec SGBD avancé (MySQL/PostgreSQL).
  • Protection contre les injections SQL.

Jalon 3 : Développement du Client Lourd

Le Jalon 3 a pour objectif de développer une application client lourd permettant la gestion des contrats de maintenance et l'export des données clients en format XML/PDF.

Fonctionnalités développées :

  • Génération de fichiers XML : Intégration des données des matériels achetés par les clients (date d'installation, date d'échéance du contrat de maintenance) pour les systèmes d'information des clients.
  • Couverture des matériels par contrat de maintenance : Possibilité pour le gestionnaire de couvrir par un contrat de maintenance les matériels déjà vendus, avec gestion des dates de renouvellement et de fin de contrat.
  • Génération de fichiers PDF de relance : Création automatique de courriers de relance pour les clients dont les contrats expirent sous 60 jours.

Contraintes techniques :

  • Couche données :
    • Dictionnaire de données détaillé.
    • Modèle conceptuel de données (MCD) avec extensions Merise 2.
    • Modèle relationnel.
    • SGBD avancé (MySQL/PostgreSQL) avec contraintes de domaine, PL/SQL, triggers, procédures stockées, événements.
    • Script de création de la base et jeu d'essai conséquent.
    • Implémentation de la BDD dans deux environnements différents (ex: Windows/MySQL et Linux/PostgreSQL).
  • Application client lourd :
    • Diagramme de classe UML.
    • Code commenté et documenté (format JavaDoc).
    • Gestion fine des erreurs.
    • Documentation technique au format HTML.
    • Diagramme de cas d'utilisation et descriptions textuelles.
    • Maquettage des IHM (composants nommés).
    • Tests unitaires et rapport de tests.
  • Règles de développement :
    • Noms de classes en notation Pascal (ex: LigneCommande).
    • Noms de méthodes, paramètres et variables en notation Camel (ex: uneQte).
    • Documentation JavaDoc pour chaque méthode publique.
    • Privilégier la lisibilité à la concision.
    • Caractères alphanumériques uniquement pour les identifiants.

Exemple de code :

/** * Génère un fichier XML pour un client donné. * @param clientId L'identifiant du client * @return Le chemin du fichier XML généré */ public String genererFichierXML(int clientId) { // Récupération des données du client et de ses contrats Client client = clientDao.findById(clientId); List contrats = contratDao.findByClient(client); // Création du document XML DocumentBuilderFactory factory = DocumentBuilderFactory.newInstance(); DocumentBuilder builder = factory.newDocumentBuilder(); Document doc = builder.newDocument(); Element root = doc.createElement("Client"); doc.appendChild(root); // Ajout des informations du client Element nom = doc.createElement("Nom"); nom.appendChild(doc.createTextNode(client.getNom())); root.appendChild(nom); // Ajout des contrats Element contratsElement = doc.createElement("Contrats"); for (Contrat contrat : contrats) { Element contratElement = doc.createElement("Contrat"); contratElement.setAttribute("id", String.valueOf(contrat.getId())); contratElement.setAttribute("dateFin", contrat.getDateFin().toString()); contratsElement.appendChild(contratElement); } root.appendChild(contratsElement); // Écriture du fichier XML String filePath = "client_" + clientId + ".xml"; TransformerFactory transformerFactory = TransformerFactory.newInstance(); Transformer transformer = transformerFactory.newTransformer(); DOMSource source = new DOMSource(doc); StreamResult result = new StreamResult(new File(filePath)); transformer.transform(source, result); return filePath; }
