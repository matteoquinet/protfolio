Stage chez 2pace – Développement Salesforce

Du 19 mai au 27 juin 2026 – Tuteur : Grégory Quinet

Présentation de l’entreprise

2pace est une entreprise spécialisée dans le développement de solutions digitales sur mesure, notamment dans l’écosystème Salesforce. Elle accompagne ses clients dans la transformation digitale de leurs processus métiers grâce à des applications performantes, modulaires et évolutives.

L’équipe allie expertise technique, méthodologie agile et écoute client pour livrer des solutions adaptées aux besoins spécifiques de chaque projet.

Objectif du stage

L’objectif principal de mon stage était de contribuer au développement d’une application Salesforce conçue pour être entièrement customisable par les utilisateurs finaux ou les administrateurs, sans intervention technique lourde.

Cette application vise à offrir une interface modulable, adaptable à différents contextes métiers, tout en respectant les contraintes et bonnes pratiques de la plateforme Salesforce.

Lightning Web Components Apex JSON Git Méthode Agile

Missions réalisées

Étude et prise en main de Salesforce

  • Découverte de l’environnement Salesforce (Lightning, Apex, Visualforce, Flows)
  • Compréhension de l’architecture orientée composant et de la logique declarative vs. programmatique
  • Formation sur Trailhead (plateforme d’apprentissage Salesforce)

Développement de composants customisables

  • Création de composants Lightning Web Components (LWC) dynamiques et réutilisables
  • Mise en place de paramètres dynamiques (labels, champs, filtres) via des fichiers JSON
  • Intégration de règles conditionnelles pour modifier l’affichage ou le comportement des modules

Participation à la conception fonctionnelle

  • Réflexion sur les types de personnalisations possibles (UI, logique métier, intégration de données)
  • Collaboration avec l’équipe produit pour définir un modèle de configuration simple mais puissant (ex. : JSON de configuration)

Tests et documentation

  • Rédaction de tests unitaires en Apex
  • Documentation technique sur les modules développés
  • Contribution à un guide utilisateur sur la personnalisation de l’application

Exemple de code : Gestion du JSON

{ "index": "2", "type": "layout", "style": "default", "nbcol": 1, "content": [ { "index": "2_1", "type": "section", "style": "error", "title": "section 1", "opened": true, "content": [...] } ] }

Fonctionnalités clés développées

  • Gestion dynamique de l’arborescence des composants
  • Sauvegarde et rafraîchissement des données après modification
  • Personnalisation de l’interface via JSON

Compétences développées

Techniques

Apex, LWC, configuration Salesforce, Git, gestion des métadonnées

Méthodologiques

Travail en méthode Agile (daily meetings, user stories), gestion de versions

Fonctionnelles

Compréhension des besoins clients, logique de configuration, UX orientée utilisateur

Bilan et perspectives

Ce stage chez 2pace a été une expérience enrichissante, tant sur le plan technique que professionnel. J’ai pu contribuer à un projet concret, dans un environnement stimulant et collaboratif.

La possibilité de travailler sur une application "low-code / no-code" m’a sensibilisé aux enjeux de flexibilité et d’autonomie des utilisateurs dans les solutions logicielles modernes.

À l’issue de ce stage, je me sens mieux préparé à travailler sur des projets Salesforce ou à évoluer dans des environnements d’intégration d’ERP/CRM. J’envisage de poursuivre dans cette voie, soit par une alternance, soit par un premier emploi dans le domaine du développement ou du consulting Salesforce.
